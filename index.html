Git 소개 및 사용법

1. Git이란?

Git은 분산 버전 관리 시스템(Distributed Version Control System, DVCS)입니다. 소프트웨어 개발 프로젝트의 소스 코드 변경 사항을 추적하고, 여러 개발자들이 동시에 협업할 수 있도록 돕는 도구입니다.

주요 특징:

  • 분산형 구조: 중앙 서버 없이 각 개발자의 로컬 저장소에 전체 프로젝트 히스토리가 저장됩니다. 이는 네트워크 연결 없이도 작업이 가능하며, 중앙 서버 장애 시에도 데이터 손실 위험이 적다는 장점이 있습니다.
  • 빠른 성능: 대부분의 작업이 로컬에서 이루어지므로 매우 빠릅니다.
  • 강력한 브랜칭 및 머징: 쉽고 빠르게 브랜치를 생성하고 병합할 수 있어 다양한 실험과 기능 개발이 용이합니다.
  • 데이터 무결성: 모든 데이터는 SHA-1 해시로 암호화되어 저장되므로, 데이터 손상으로부터 안전합니다.

2. Git 설치

운영체제에 맞는 Git을 설치하는 방법을 안내합니다.

Windows

공식 웹사이트에서 설치 파일을 다운로드하여 설치합니다.

  • Git for Windows 다운로드

설치 시 특별한 설정 변경 없이 기본 옵션으로 진행해도 무방합니다.

macOS

macOS에서는 Homebrew를 통해 쉽게 설치할 수 있습니다. 터미널을 열고 다음 명령어를 입력하세요.

brew install git

또는 Xcode Command Line Tools를 설치하면 Git이 함께 설치됩니다.

xcode-select --install

Linux (Debian/Ubuntu)

터미널을 열고 다음 명령어를 입력하여 설치합니다.

sudo apt update
sudo apt install git

설치 확인

설치 후 터미널 또는 명령 프롬프트에서 다음 명령어를 입력하여 Git이 올바르게 설치되었는지 확인합니다.

git --version

버전 정보가 출력되면 성공적으로 설치된 것입니다.

3. Git 기본 사용법

Git을 사용하여 프로젝트를 관리하는 기본적인 명령어들을 소개합니다.

3.1. 사용자 정보 설정

Git 커밋에 사용될 사용자 이름과 이메일 주소를 설정합니다. 이 정보는 각 커밋에 기록됩니다.

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

--global 옵션은 해당 컴퓨터의 모든 Git 저장소에 적용됩니다. 특정 저장소에만 적용하려면 --global을 제외하고 해당 저장소 디렉토리에서 명령어를 실행합니다.

3.2. 새 Git 저장소 생성 (git init)

기존 프로젝트 폴더를 Git 저장소로 만들거나, 새로운 빈 Git 저장소를 생성할 때 사용합니다.

cd my_project_folder
git init

이 명령어를 실행하면 .git이라는 숨김 폴더가 생성되며, 이 폴더 안에 Git 저장소의 모든 정보가 저장됩니다.

3.3. 원격 저장소 복제 (git clone)

GitHub와 같은 원격 저장소에 있는 프로젝트를 로컬 컴퓨터로 가져올 때 사용합니다.

git clone [원격 저장소 URL]

예시:

git clone https://github.com/user/repo.git

3.4. 변경 사항 추적 (git add)

작업 디렉토리에서 수정한 파일들을 스테이징 영역(Staging Area)에 추가합니다. 스테이징 영역에 있는 파일들만 커밋에 포함됩니다.

  • 특정 파일 추가:
    • git add [파일명]
  • 모든 변경된 파일 추가:
    • git add .

3.5. 변경 사항 기록 (git commit)

스테이징 영역에 있는 변경 사항들을 로컬 저장소에 영구적으로 기록합니다. 각 커밋은 프로젝트의 특정 시점 스냅샷을 나타냅니다.

git commit -m "커밋 메시지"

-m 옵션 뒤에 해당 커밋의 내용을 설명하는 메시지를 작성합니다. 좋은 커밋 메시지는 나중에 변경 이력을 파악하는 데 큰 도움이 됩니다.

3.6. 저장소 상태 확인 (git status)

현재 작업 디렉토리와 스테이징 영역의 상태를 보여줍니다. 어떤 파일이 수정되었는지, 스테이징되었는지 등을 확인할 수 있습니다.

git status

3.7. 커밋 이력 확인 (git log)

현재 브랜치의 커밋 이력을 시간 순서대로 보여줍니다.

git log

다양한 옵션을 통해 출력 형식을 조절할 수 있습니다 (예: git log --oneline --graph).

3.8. 원격 저장소로 푸시 (git push)

로컬 저장소의 커밋들을 원격 저장소로 업로드합니다. 주로 origin이라는 이름의 원격 저장소의 main 또는 master 브랜치로 푸시합니다.

git push origin main

최초 푸시 시에는 -u 옵션을 사용하여 로컬 브랜치와 원격 브랜치를 연결할 수 있습니다.

git push -u origin main

3.9. 원격 저장소에서 풀 (git pull)

원격 저장소의 변경 사항을 로컬 저장소로 가져와 병합합니다. 다른 개발자가 푸시한 내용을 내 로컬 작업에 반영할 때 사용합니다.

git pull origin main
